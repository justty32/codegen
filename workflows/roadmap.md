# 路線圖（roadmap）— 工作流入口

← [INDEX](../INDEX.md)｜[CLAUDE.md](../CLAUDE.md)

記一件**確定會做、但不確定何時**的事。比 [idea](idea.md) 更篤定（已決定要做），但還沒到 [spec](spec.md)（還沒展開設計）。

## 用法

- 把確定要做的事追加到下面，一條一行；可附一句「為什麼要做 / 大致範圍」。
- 要開始討論做法 → 進 [spec](spec.md)。
- 取消的就刪掉。

規劃管線：idea → **roadmap（本工作流）** → spec → plan → build。

## 膨脹時

項目多到需要分類（依領域 / 優先序）→ 升級成資料夾（`roadmap/README.md` + 各分類檔）。
